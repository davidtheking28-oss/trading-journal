01
Interactive Brokers

הגדרת Flex Query

ℹ️
מה זה Flex Query? שירות של IBKR שמאפשר להוריד היסטוריית עסקאות בפורמט XML. אנחנו משתמשים בו כדי לייבא את העסקאות שלך אוטומטית.
1
כנס לחשבון IBKR
עבור ל-client.ibkr.com ובצע התחברות עם שם המשתמש והסיסמה שלך.
2
כנס ל-Reports → Flex Queries
בתפריט העליון לחץ על Reports ← Flex Queries. זה בדרך כלל נמצא תחת לשונית "Performance & Reports".
3
צור Flex Query חדש לעסקאות
לחץ על Create ← Trade Confirmation Flex Query. תן לו שם כלשהו (למשל "Trading Journal").
✅
שדות חובה שצריך לסמן: Symbol, Buy/Sell, Quantity, Price, Commission, TradeDate, OrderType, AssetCategory, Open/CloseIndicator
4
הגדר טווח תאריכים
בחר Date Range → Last 30 Calendar Days (או מה שמתאים לך). פורמט: XML. לחץ Save.
5
העתק את Query ID
לאחר השמירה, תראה את ה-Query ברשימה עם מספר ID. העתק אותו.
Query ID — דוגמה
1462900
6
קבל Flex Web Service Token
לך ל-Manage Account → Settings → Flex Web Service. הפעל את השירות ולחץ Generate Token. שמור את ה-Token — הוא מוצג פעם אחת בלבד.
⚠️
שמור את ה-Token! IBKR מציג אותו פעם אחת. אם שכחת, תצטרך לייצר חדש.
Token — נראה כך
1234567890abcdef1234567890abcdef
7
הזן ב-Trading Journal
חזור לאפליקציה → Settings → Interactive Brokers. הדבק את ה-Token ואת ה-Query ID ולחץ Save.
02
Bybit

יצירת API Key

ℹ️
הרשאות נדרשות: צריך API Key עם הרשאת Read Only בלבד — אנחנו לא שולחים פקודות, רק קוראים היסטוריית עסקאות.
1
כנס לחשבון Bybit
עבור ל-bybit.com והתחבר לחשבון שלך.
2
כנס ל-API Management
לחץ על תמונת הפרופיל → My Profile → API. לחלופין: Settings → API Management.
3
צור API Key חדש
לחץ Create New Key. בחר System-generated API Keys.
4
הגדר הרשאות Read Only
בשדה Permissions, סמן רק:
  • Trade History — Read
  • Order — Read
  • Position — Read
⚠️
אל תסמן Withdraw! לשם הקרא בלבד אין צורך בהרשאת משיכה.
5
שמור API Key ו-Secret
לאחר הלחיצה על Submit, Bybit יציג את ה-API Key וה-Secret פעם אחת. שמור אותם.
API Key
abc123XYZ...
API Secret
••••••••••••••••
6
הזן ב-Trading Journal
חזור לאפליקציה → Settings → Bybit. הדבק את ה-API Key וה-Secret ולחץ Save.
03

סנכרון ובדיקה

1
לחץ Sync
לאחר שמירת ה-Token/API Key, לחץ על כפתור Sync. הייבוא יכול לקחת 10–30 שניות.
2
בדוק את הלוג
תראה הודעות כמו "נמצאו 47 עסקאות" ו-"יובאו 23 עסקאות חדשות". אם יש שגיאה — ראה פתרון בעיות למטה.
3
בדוק בטבלת העסקאות
עבור ללשונית Stocks או Crypto ווידא שהעסקאות מופיעות עם הנתונים הנכונים.
💡
הסנכרון הוא חד-כיווני — רק קריאה מהברוקר. לא ניתן לבצע פקודות מסחר דרך האפליקציה.
04

פתרון בעיות נפוצות

❌ "Token לא תקין" או "Unauthorized"
בדוק שהפעלת את Flex Web Service ב-IBKR (Manage Account → Settings → Flex Web Service). ייצר Token חדש אם צריך.
❌ "Query ID לא נמצא"
ודא שה-Query ID הוא מספרי בלבד (לדוגמה 1462900) ושה-Query נשמר כ-Trade Confirmation Flex Query.
❌ "0 עסקאות נמצאו" למרות שיש עסקאות
בדוק את טווח התאריכים שהגדרת ב-Flex Query. ב-IBKR, הגדר Last 30 Calendar Days או טווח מותאם שמכסה את תקופת המסחר שלך.
❌ עסקאות מופיעות ב-Stocks אבל ללא Stop Loss
ה-Stop Loss לא מגיע מ-IBKR — צריך להזין ידנית לאחר הייבוא. ה-Flex Query לא מכיל פקודות עתידיות, רק ביצועים.
❌ Bybit — "Invalid API Key"
ודא שה-API Key לא פג תוקף וש-IP שלך לא חסום. ב-Bybit ניתן לראות מצב ה-Key תחת My Profile → API.
❌ עסקאות כפולות לאחר Sync
האפליקציה בודקת כפילויות לפי ID. אם קרה — עדכן את הדף ובצע Sync מחדש. אם הבעיה חוזרת, צור קשר.
מוכן? בוא נחבר את הברוקר

כנס לאפליקציה → Settings → בחר את הברוקר שלך ועקוב אחרי המדריך.
