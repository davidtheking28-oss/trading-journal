## מה השתנה

### 1. פיצול הקובץ (ביצועים)
הקובץ המקורי (867KB) פוצל לשלושה:

| קובץ | גודל | תפקיד |
|------|------|--------|

מעכשיו הדפדפן שומר את ה-CSS וה-JS ב-cache בין ביקורים — טעינות חוזרות מורידות רק ~131KB במקום 867KB. בנוסף, ה-service worker עודכן (`tj-v30`) כך שהקבצים החדשים נשמרים גם לעבודה offline.

**חשוב לדיפלויים עתידיים:** כשתשנה את `styles.css` או `app.js`, העלה את מספר הגרסה בשני מקומות:
- ב-`dashboard.html`: `styles.css?v=21` → `?v=22` (וכנ"ל ל-app.js)
- ב-`sw.js`: עדכן את שתי השורות ב-PRECACHE ואת שם ה-CACHE (`tj-v30` → `tj-v31`)

### 2. אבטחה — Bybit API Secret לא חוזר יותר לדפדפן
- ה-`select` של ההגדרות כבר לא מבקש את `bybit_api_secret`
- שדה ה-Secret מציג placeholder "•••• (שמור בשרת)" במקום את הסוד עצמו
- `bybitSave()` עודכן: אפשר לעדכן את ה-Key בלי להקליד מחדש את ה-Secret (הוא write-only — נשלח רק כשמזינים אותו)

**צעד נדרש ב-Supabase (חד-פעמי):** הקוד כבר לא מבקש את העמודה, אבל כל עוד ההרשאות פתוחות — כל מי שמחזיק session יכול לבקש אותה ידנית. כדי לחסום ברמת ה-DB, הרץ ב-SQL Editor:

```sql
-- שלב 1: ראה את כל העמודות בטבלה
select column_name from information_schema.columns
where table_schema = 'public' and table_name = 'user_settings';

-- שלב 2: בטל SELECT כללי והענק רק על העמודות הלא-סודיות
revoke select on public.user_settings from authenticated, anon;
grant select (
  user_id, groq_api_key, groq_inv_key, finnhub_key, fmp_key, av_key,
  flex_token, flex_query_id, portfolio_total, bybit_api_key
  -- הוסף כאן כל עמודה נוספת שהופיעה בשלב 1, חוץ מ-bybit_api_secret
) on public.user_settings to authenticated;
```

ה-edge function של Bybit ממשיך לקרוא את הסוד כרגיל (הוא רץ עם service_role).
שקול לעשות את אותו הדבר גם ל-`flex_token` של IBKR אם ה-edge function הוא היחיד שצריך אותו.

**המלצה נוספת:** ודא שהמפתחות ב-Bybit וב-IBKR מוגדרים **Read-Only** — כך גם דליפה לא מאפשרת מסחר או משיכה.

### 3. אבטחה — תפוגה ל-hash ישן של PIN
PIN ישן (SHA-256 בלי salt, מלפני המעבר ל-PBKDF2) יתקבל רק עד **1.9.2026**. אחרי התאריך הזה ה-hash הישן נמחק והמשתמש מגדיר PIN מחדש (שנשמר כבר ב-PBKDF2 עם 200K איטרציות). מי שמתחבר לפני התאריך — עובר מיגרציה אוטומטית כרגיל.

### 4. פיצ'רים חדשים בעמוד Statistics
- **Max Drawdown** — KPI חדש: הירידה המקסימלית של עקומת ההון מהשיא, לפי תאריכי סגירה. המספר החשוב ביותר לניהול סיכונים שלא היה לך.
- **גרף Drawdown (Underwater)** — מראה כמה רחוק היית מהשיא בכל נקודת זמן. מאפשר לזהות תקופות קשות ולבדוק אם ה-drawdown מתארך.
- **R-Multiple Distribution** — היסטוגרמה של כל העסקאות הסגורות לפי כפולות R (מ-"מתחת ל--2R" עד "מעל +5R"). לסוחר בשיטת מינרוויני זה הגרף שמראה אם אתה באמת חותך הפסדים ב--1R ונותן לרווחים לרוץ — או שההתפלגות מספרת סיפור אחר.

שני הגרפים מכבדים את הסינון הקיים (תקופה / מניות / קריפטו) ואת מצב הפרטיות, ומציגים הודעה ידידותית כשאין מספיק עסקאות.
